# 핀모아 (PinMoa) — 서비스 기획서

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 핀모아 (PinMoa) |
| 슬로건 | 릴스에서 발견, 지도에 저장, 친구랑 같이 |
| 플랫폼 | iOS / Android (모바일 앱) |
| 타겟 | 10~20대, SNS로 장소를 공유하는 친구·커플 그룹 |

---

## 2. 문제 정의

### 현재 상황
- 친구끼리 인스타그램 릴스·틱톡을 통해 맛집·카페·명소를 활발히 공유함
- "여기 가보자"며 링크를 주고받지만, 막상 약속 잡을 때는 어디서 봤는지 기억나지 않음
- DM, 카카오톡 등 여러 채널에 흩어진 링크를 다시 뒤져야 하는 번거로움 발생

### 핵심 페인포인트
1. **발견 → 저장 단절**: 릴스를 보고 "가고 싶다"고 생각해도 장소를 직접 찾아서 저장하는 과정이 귀찮아 흐지부지됨
2. **공유 → 계획 단절**: 친구에게 보낸 링크가 메신저에 묻히고, 약속 잡을 때 다시 찾기 어려움
3. **나만의 맛집 지도 부재**: 내가 저장한 곳과 친구가 저장한 곳을 한 화면에서 비교할 수 없음

---

## 3. 서비스 핵심 가치

> **릴스 링크 하나로 장소를 자동 저장하고, 친구들과 지도를 함께 만들어 간다.**

콘텐츠 루프:
```
친구가 릴스 발견 → 링크 저장 (AI 자동 분석)
→ 스페이스에 함께 모임 → 내가 다녀옴
→ 폴라로이드 후기 업로드 → 친구들이 보고 가고 싶어짐 → 반복 🔁
```

## 5. 기능 명세서

---

### F-01. SNS 링크 AI 장소 추출

**설명**
사용자가 인스타그램 릴스(또는 틱톡) URL을 붙여넣으면, AI가 영상의 캡션·해시태그·계정 정보를 분석해 장소 후보를 제시하고, 사용자가 확인 후 지도에 저장한다.

**사용자 흐름**
1. 하단 `+` 버튼 또는 "링크 붙여넣기" CTA 탭
2. URL 입력 필드에 릴스 링크 붙여넣기
3. "가져오기" 버튼 탭 → 로딩 화면 (캡션·해시태그·계정명 파싱 중)
4. AI가 분석한 장소 후보 최대 3개 표시
   - 최상위 후보: "가장 정확한 후보예요" 배지
   - 나머지: 이름·거리·카테고리 표시
5. 사용자가 맞는 장소 선택
6. 공유 스페이스 선택 (선택 사항)
   - 내 스페이스에는 **항상 자동 저장** (체크 해제 불가)
   - 참여 중인 공유 스페이스 목록 표시 → 복수 선택 가능
   - 예) A, B, C 스페이스 중 A·B만 선택 → 내 스페이스 + A + B에 동시 등록
7. "저장하기" 버튼 탭 → 선택한 모든 스페이스에 즉시 등록

**입력 데이터**
- Instagram Reels URL / TikTok URL

**AI 분석 대상**
- 캡션 텍스트 (장소명, 지역명 키워드)
- 해시태그 (예: `#성수맛집`, `#파스타`)
- 계정명·위치 태그

**출력 데이터**
- 장소 후보 목록 (이름, 카테고리, 주소, 거리, 정확도 순위)

**예외 처리**
- 장소 정보를 특정할 수 없는 경우: "장소를 찾지 못했어요. 직접 검색해 주세요" 안내
- 유효하지 않은 URL: 입력 필드 에러 표시
- 네트워크 오류: 재시도 버튼 제공

### F-02. 스페이스 (내 스페이스 + 공유 지도)

**설명**
모든 장소는 별도의 "내 지도" 탭 없이 스페이스 안에서 관리된다. 회원가입 시 **내 스페이스**가 자동 생성되며, 이곳에 링크로 저장한 장소들이 쌓인다. 친구와 함께 만드는 공유 스페이스는 추가로 생성할 수 있다.

**스페이스 개념**

| 구분 | 설명 |
|------|------|
| 내 스페이스 | 가입 시 자동 생성. 나 혼자 사용하는 기본 공간. 삭제 불가 |
| 공유 스페이스 | 친구를 초대해 함께 만드는 공간. 복수 생성 가능 |

**스페이스 목록 탭**
- 최상단: 내 스페이스 카드 (항상 고정)
- 이하: 내가 참여 중인 공유 스페이스 카드 목록
- 카드 정보: 스페이스 이름, 이모지, 저장 장소 수, 멤버 아바타, 최신 후기 미리보기
- 알림 배지: "방금 지윤이 후기 올림" 등 실시간 활동
- 우측 상단 `+` 버튼으로 새 공유 스페이스 생성

**내 스페이스 상세**
- 나만 있는 스페이스 (멤버 = 나 1명)
- 장소 검색, 리스트/지도 토글 동일하게 제공
- "링크 붙여넣기" 퀵 CTA 배너
- 저장한 장소 수 요약 표시
- 장소 카드에서 "공유 스페이스에 추가" 기능 (나중에 특정 스페이스에 추가하고 싶을 때)

**장소 저장 시 스페이스 선택 규칙**
- 내 스페이스: 항상 자동 저장 (해제 불가)
- 공유 스페이스: 저장 시 목록에서 복수 선택 → 동시에 등록됨
- 저장 후에도 장소 상세에서 추가 공유 스페이스 선택 가능

**공유 스페이스 생성 (2단계)**

Step 1 — 이름 설정
- 스페이스 이름 입력
- 대표 이모지 선택

Step 2 — 친구 초대
- 초대 링크 복사 (링크 받은 친구가 합류하면 지도 자동 병합)
- 이름·아이디로 친구 검색 및 직접 초대
- 초대 상태 표시 (초대함 / + 초대)

**스페이스 상세 (공통)**
- 뷰 탭: 지도 / 리스트 / 로그
- 멤버 아바타 목록 + 멤버 수 (내 스페이스는 "나" 1명)
- 리스트: 겹침 많은 순 정렬 (기본, 공유 스페이스) / 최근 저장 순 (내 스페이스 기본)

**공유 스페이스 참여**
- 초대 링크를 통해 딥링크로 진입
- 참여 시 내 스페이스 장소가 해당 공유 스페이스에 자동 반영

### F-03. 회원 가입

**설명**
서비스 진입을 위한 계정 생성 및 초기 설정. 가입 완료 시 내 스페이스가 자동 생성된다.

**가입 흐름**
1. 소셜 로그인 (카카오 / 애플)
2. 닉네임 입력
3. 프로필 사진 설정 (선택)
4. 가입 완료 → 내 스페이스 자동 생성 → 홈(스페이스 탭)으로 이동

**규칙**
- 내 스페이스는 가입 시 1회 자동 생성, 삭제 불가
- 닉네임은 앱 내 친구 검색에 사용됨

---

### F-04. 장소 상세 페이지

**설명**
장소를 탭하면 진입하는 상세 화면. 기본 정보 외에 이 장소가 등장한 릴스 목록과 친구 후기를 보여준다.

**구성 요소**
- 히어로 이미지 (장소 대표 이미지)
- 장소명, 별점, 카테고리, 주소, 거리
- 저장한 멤버 수 / 영업 상태 (영업중 · 마감 시간)
- 이 장소를 저장한 릴스 가로 스크롤 (웹뷰로 릴스 재생)
- 친구들 후기 피드
- "다녀왔어요, 후기 올리기" CTA 버튼

**릴스 웹뷰**
- 썸네일 카드 탭 시 앱 내 웹뷰로 원본 릴스 재생
- 계정명, 조회수 표시

---

### F-05. 후기 업로드

**설명**
장소를 다녀온 후 폴라로이드 스타일로 사진 + 한 줄 후기를 남긴다. 업로드된 후기는 해당 스페이스에 영구적으로 누적되며 스페이스 멤버들에게 노출된다.

**업로드 흐름**
1. 장소 상세 → "후기 올리기" 탭
2. 폴라로이드 프레임 내 사진 업로드
3. 한 줄 후기 텍스트 입력 (손글씨 폰트 표현)
4. "올리기" 탭 → 해당 장소가 등록된 스페이스 멤버 전체에게 공유

**후기 노출**
- 장소 상세 "친구들 후기" 섹션에 표시
  - 프로필 아바타, 이름, 작성 시간
  - 한 줄 텍스트 + 사진 썸네일
- 스페이스 리스트의 해당 장소 카드 하단 인라인 표시
- 스페이스 카드에 "방금 OOO이 후기 올림" 실시간 배지

---

### F-06. 알림

**설명**
스페이스 내 활동 알림을 푸시 및 알림 탭에서 제공한다.

**알림 유형**

| 트리거 | 알림 내용 |
|--------|-----------|
| 친구가 같은 장소 저장 | "OOO이 [장소명]을 저장했어요 — 겹침!" |
| 스페이스 멤버가 후기 업로드 | "OOO이 [장소명] 후기를 올렸어요 📸" |
| 스페이스 초대 수락 | "OOO이 [스페이스명]에 합류했어요" |
| 내가 저장한 장소에 후기 | "저장한 [장소명]에 새 후기가 달렸어요" |

## 6. 화면 목록 (스크린 플로우)

```
[스페이스 탭]
  ├── 스페이스 목록
  │     ├── 내 스페이스 (최상단 고정, 가입 시 자동 생성)
  │     │     ├── 리스트 뷰
  │     │     │     └── 장소 상세 → 후기 올리기
  │     │     └── 지도 뷰
  │     │           └── 장소 상세 (하단 카드)
  │     └── 공유 스페이스 (친구 초대 생성)
  │           ├── 스페이스 상세 — 리스트 뷰
  │           ├── 스페이스 상세 — 지도 뷰
  │           └── 스페이스 상세 — 로그 뷰
  └── 공유 스페이스 생성 (Step 1 → Step 2)

[+ 버튼]
  └── 링크 입력 → 로딩 → 장소 후보 선택 → 저장할 스페이스 선택 → 저장 완료

[알림 탭]

[프로필 탭]
```

## 7. 기술 요구사항 (백엔드 관점)

| 항목 | 내용 |
|------|------|
| SNS 링크 파싱 | Instagram Graph API 또는 크롤링으로 캡션·해시태그 추출 |
| AI 장소 추출 | LLM (Claude / GPT)으로 텍스트 → 장소 후보 추론 |
| 장소 검색 | Kakao Map API / Google Places API로 후보 매칭 |
| 지도 렌더링 | Kakao Map SDK / Naver Map SDK |
| 실시간 알림 | FCM (Firebase Cloud Messaging) |
| 스페이스 공유 | 딥링크 (Universal Link / App Link) |
| 후기 저장 | 스페이스에 영구 누적 |
